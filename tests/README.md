# Tests

Unit tests and validation checks for core logic.

Purpose:
Ensure correctness of:
- mathematical functions
- simulations
- transformations
- financial metrics

Guidelines:
- Focus on testing core logic and invariants.
- Prefer small, deterministic tests.
- Avoid testing plotting or exploratory code.

Examples:
- test_returns.py
- test_volatility.py
- test_gbm.py
