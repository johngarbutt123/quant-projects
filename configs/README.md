# Configs

Configuration files for experiments, simulations, and models.

Typical contents:
- YAML / JSON / TOML files defining:
  - model parameters
  - simulation settings
  - experiment definitions
  - backtest configurations

Example:
- gbm.yaml
- factor_model.toml
- backtest.json

Purpose:
Keep parameters separate from code to ensure:
- reproducibility
- easy experiment tracking
- clean separation of logic and configuration
